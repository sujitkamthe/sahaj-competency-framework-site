## Mindset

"I am learning how things are done, why they are done this way, and how to contribute safely."

**The question you're trusted to answer:** "How do I do this specific thing?"

## Nature of Impact

- Individual
- Task-focused
- High learning velocity, low risk-taking

## Success Looks Like

- Becoming dependable
- Learning Sahaj's engineering and consulting culture
- Asking good questions early

---

# Technical Delivery

## Expectations

- Understands core programming constructs and language idioms
- Learns frameworks, libraries, and tools used in the project
- Sets up local development environment with help
- Implements small, well-defined tasks with guidance
- Writes correct but often linear or naive solutions
- Learns and follows coding standards and formatting rules
- Understands basic Git workflows (commit, pull, push, rebase)
- Uses logs and debuggers to investigate issues
- Writes unit tests when explicitly asked
- Fixes bugs with clear reproduction steps
- Responds constructively to PR feedback
- Improves code based on review comments
- Understands system architecture only at a conceptual level
- Avoids obvious regressions when making changes
- Learns basic security hygiene (secrets, credentials)

## Self-Assessment

- Can I complete a small task without repeated clarification?
- Do I understand what my code changes affect?
- Do I rely heavily on others to debug issues?

# Consulting

## Expectations

- Listens carefully to problem statements without jumping to solutions
- Seeks clarification when requirements are unclear or inconsistent
- Asks basic questions to understand context
- Understands assigned tasks within the immediate feature scope
- Relies on others to define approach
- Accepts given constraints without challenging them
- Learns client or domain vocabulary gradually
- Escalates ambiguity rather than resolving it

## Self-Assessment

- Do I mostly wait for clarity or actively seek it?
- Can I explain the problem in my own words?
- Do I understand why this work matters?
